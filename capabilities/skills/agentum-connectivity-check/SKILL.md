# Agentum 连通性检查 Skill

## 用途

用于验证 Agentum 能力登记、租户能力池分配、智能体模板引用和运行时 Skill 装配链路是否连通。

## 输入

- `topic`：需要检查或总结的业务主题。
- `context`：可选业务上下文。

## 输出

返回一段结构化中文摘要：

```json
{
  "summary": "检查结论",
  "missingInformation": ["仍需补充的信息"],
  "nextAction": "建议下一步"
}
```

## 使用边界

- 只用于本地和测试环境验证能力链路。
- 不读取外部系统、不处理敏感凭证、不代替正式业务 Skill。
- 如果输入缺少上下文，应明确列出缺失信息，而不是编造业务事实。

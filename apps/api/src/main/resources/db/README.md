# db

数据库迁移脚本放在 `migration` 目录中，当前使用 Flyway 管理。开发阶段允许重建本地数据库，因此迁移脚本按用途拆分为真实结构和开发演示数据，便于后续删表、删字段和重建环境。

要求：

- 数据库结构变更必须新增迁移脚本并随代码提交，放入 `migration/schema/`。
- 本地演示账号、演示租户、演示能力和临时兼容数据放入 `migration/devdata/`，不得和真实表结构混写。
- 迁移文件命名使用 `VyyyyMMddHHmm__description.sql`。
- Flyway 版本号在 `schema` 和 `devdata` 两个目录间也必须全局唯一，避免本地同时扫描时报重复版本。
- 关键表和字段必须写中文注释，便于后续排查和交接。
- 默认配置只扫描 `schema`；`local` profile 同时扫描 `schema` 和 `devdata`，用于本地联调。

## 目录说明

```text
db/migration/
  schema/   真实表结构、索引、约束、结构性数据迁移
  devdata/  本地开发账号、演示租户、演示能力、开发期兼容清理
```

开发阶段如果某张表或字段已经不符合当前功能心智，可以直接通过新的结构迁移调整或删除；优先保证模型清晰、迁移可重放和本地环境可快速重建，不为了兼容早期本地库长期保留过时字段。

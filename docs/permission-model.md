# 权限模型

## 1. 设计目标

Agentum 的权限设计需要覆盖平台管理、资产治理、工作流执行和外部系统调用。

要求：

- 权限从第一版开始内建，不后补。
- 角色用于控制人，资源权限用于控制资产。
- MCP、凭证、生产交付节点必须重点保护。
- 所有敏感操作必须审计。

## 2. 权限模型

推荐采用：

```text
认证身份 + 组织 / 空间归属 + RBAC + 资源级权限 + 审计日志
```

认证身份负责确认“是谁”，组织 / 空间归属负责确认“属于哪里”，RBAC 负责角色，资源级权限负责具体资产。

## 3. 身份、组织与空间

### 3.1 用户身份

用户至少包含：

- 用户 ID
- 登录账号
- 显示名
- 邮箱
- 手机号
- 头像
- 账号状态：启用、停用、锁定、待激活
- 最近登录时间
- 默认组织
- 默认空间
- 默认角色

第一阶段可以先支持账号密码登录，接口和页面需要预留 SSO、MFA、忘记密码、会话过期和退出全部设备入口。

### 3.2 组织

组织是企业或业务主体边界。

组织至少包含：

- 组织 ID
- 组织名称
- 组织状态
- 组织管理员
- 成员数量
- 空间数量
- 创建时间

组织管理员可以管理组织内空间、成员邀请和组织级策略，但不能绕过系统管理员管理全局配置。

### 3.3 空间

空间是资源隔离边界。工作流、智能体模板、Skills、MCP、知识库、凭证和运行记录都应归属到某个空间。

空间至少包含：

- 空间 ID
- 空间名称
- 所属组织
- 空间管理员
- 成员列表
- 默认角色
- 资源配额

用户可以属于多个空间，并在不同空间拥有不同角色。

## 4. 推荐角色

| 角色 | 说明 |
| --- | --- |
| 系统管理员 | 管理全局配置、用户、空间、模型、系统级 MCP |
| 空间管理员 | 管理业务空间成员、资产和权限 |
| 流程设计者 | 创建和编辑工作流 |
| 智能体管理员 | 创建、维护和发布智能体模板 |
| 能力管理员 | 管理 Skills、MCP、知识库和工具凭证 |
| 审核人 | 处理人工审核节点 |
| 执行人 | 触发流程、填写输入、查看自己的运行结果 |
| 观察者 | 只读查看流程、资产和执行记录 |

## 6. 权限动作

基础动作：

- `read`
- `create`
- `update`
- `delete`
- `execute`
- `publish`
- `approve`
- `cancel`
- `retry`
- `manage_permission`

敏感动作：

- `manage_user`
- `manage_role`
- `manage_space`
- `manage_credential`
- `invoke_mcp`
- `invoke_sensitive_mcp`
- `download_file`
- `deliver_to_external_system`
- `view_audit_log`

## 7. 权限判断顺序

建议按以下顺序判断：

1. 用户是否登录。
2. 用户是否属于当前空间。
3. 用户角色是否具备基础动作。
4. 用户是否对目标资源有访问权限。
5. 操作是否涉及敏感能力。
6. 是否需要二次确认或人工审批。
7. 记录审计日志。

## 8. 登录与会话

登录与会话要求：

- 登录失败返回中文 message，不暴露账号是否存在的敏感细节。
- 账号停用、锁定、未分配空间时要返回明确业务提示。
- 会话过期后前端跳转到会话过期页，保留原访问路径，重新登录后可回跳。
- 退出登录必须清理前端 token、用户缓存、空间缓存和敏感页面状态。
- 高风险操作可以要求重新认证或 MFA。

## 9. 用户、组织与角色管理

用户管理要求：

- 系统管理员可查看全部用户。
- 组织管理员只能查看本组织用户。
- 空间管理员只能查看本空间成员。
- 普通用户只能查看自己的个人资料和公开组织信息。

角色管理要求：

- 系统角色由系统管理员维护。
- 空间角色由系统管理员或空间管理员维护。
- 业务角色可以绑定到流程待办、审核节点和执行范围。
- 一个用户可以在不同空间拥有不同角色。

权限策略要求：

- 角色只授予动作能力，资源权限决定具体资源是否可访问。
- 敏感动作需要单独标记，并可配置审批或二次确认。
- 权限策略变更必须审计，并记录变更前后摘要。

## 10. MCP 权限

MCP 是高风险能力，需要单独控制。

每个 MCP 服务应配置：

- 可见范围
- 可调用角色
- 可调用智能体
- 可调用工作流
- 参数白名单
- 调用频率限制
- 是否需要审批
- 是否允许生产环境调用

## 11. 凭证权限

工具凭证不允许前端直接读取明文。

要求：

- 凭证服务端加密存储。
- 前端只显示脱敏信息。
- 凭证绑定、解绑、更新必须审计。
- 生产凭证和测试凭证隔离。
- 高风险凭证需要空间管理员审批。

## 12. 审计日志

必须审计：

- 登录和登出
- 登录失败、账号锁定、会话过期
- 创建、修改、禁用用户
- 邀请成员、移除成员、调整空间角色
- 创建、修改、删除组织或空间
- 创建、修改、删除工作流
- 发布工作流
- 修改智能体提示词
- 修改 Skill 或 MCP 配置
- 调用 MCP
- 绑定或更新凭证
- 人工审核通过或驳回
- 对外交付文件或消息
- 下载敏感文件

审计字段：

- 操作人
- 操作时间
- 资源类型
- 资源 ID
- 动作
- 请求 ID
- IP
- User-Agent
- 操作前摘要
- 操作后摘要
- 结果状态

## 1. 当前阶段

当前处于 **阶段一：框架与基础治理**。

阶段一目标不是一次性做完所有智能体能力，而是先搭出可持续扩展的框架：

```text
认证与租户上下文
  -> 用户 / 部门 / 角色 / 权限基础
  -> 工作台与流程设计骨架
  -> 能力资产骨架
  -> 工作流运行和暂停恢复
  -> 审计与交付闭环
```

第一阶段闭环目标：

```text
用户输入 -> 智能体分析/追问 -> 并行子智能体获取数据 -> 合并组装 -> 人工审核 -> 文档/邮件/OA 交付
```

本轮设计决策：

- 知识库 / RAG 从第一阶段移除。
- 能力资产保留智能体模板、Skills、MCP、提示词模板和交付能力。
- 权限和审计提示从能力资产页弱化，主要放入权限管理和系统管理。
- 新增系统管理，承载租户、底层模型、全局 MCP / Skills、交付能力和多租户授权。
- 当前进度文档独立放入 `docs/progress/`，不混入长期主文档。
- 数据库使用 PostgreSQL，版本随代码通过 Flyway 迁移同步。

## 4. 功能放置建议

| 内容 | 放置位置 | 原因 |
| --- | --- | --- |
| 底层模型供应商 | 系统管理 | 平台级配置，再授权给租户使用 |
| 全局 MCP 注册 | 系统管理 | 连接能力和凭证策略属于平台治理 |
| MCP 在某租户内可用 | 系统管理 + 权限管理 | 系统给租户授权，租户再分配给角色 / 智能体 / 工作流 |
| Skills 上架 | 系统管理或能力资产 | 通用 Skills 可平台上架，业务 Skills 可租户内创建 |
| 智能体模板 | 能力资产 | 是流程设计的核心复用单元 |
| 提示词模板 | 能力资产 | 被智能体模板或节点引用，需版本化 |
| 交付能力 | 能力资产 + 系统管理 | 能力定义在资产，底层通道和租户授权在系统管理 |
| 用户、部门、角色 | 权限管理 | 直接决定谁能执行、审核、设计和发布 |
| 资源权限 | 权限管理 | 控制具体流程、资产、运行记录和审计日志 |
| 我的待办 | 业务工作台 | 表示当前用户需要处理的暂停点 |
| 运行态摘要 | 业务工作台 / 运行审计 | 表示有权限查看的运行状态，不一定需要处理 |
| 流程模板发起 | 业务工作台 | 点击后创建运行实例，进入交互式步骤，不进入设计画布 |

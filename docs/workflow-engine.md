# 工作流引擎设计

## 1. 设计目标

工作流引擎是 Agentum 的核心。它需要做到：

- 节点类型固定，组合方式灵活。
- 支持用户输入、智能体追问、人工审核等暂停点。
- 支持并行执行和合并。
- 支持条件分支和回退。
- 支持变量传递和结构化输出。
- 支持完整执行日志和审计。

## 2. 核心对象

### 2.1 WorkflowDefinition

工作流草稿或配置模板，包含节点、边和画布布局。

### 2.2 WorkflowVersion

发布后的不可变版本。每次正式运行必须绑定具体版本。

### 2.3 WorkflowRun

一次工作流运行实例。

### 2.4 NodeDefinition

节点定义，描述节点类型、配置、输入输出变量和下游连接。

### 2.5 NodeRun

节点运行记录，描述某个节点在某次运行中的状态、输入、输出、耗时和错误。

### 2.6 VariableSnapshot

变量快照，记录某一时刻流程中的变量值，便于回放和排错。

## 3. 节点类型

| 节点类型 | 说明 | 是否允许暂停 |
| --- | --- | --- |
| trigger | 流程入口 | 否 |
| user_input | 用户填写表单或上传文件 | 是 |
| agent | 智能体执行节点 | 可选 |
| parallel_group | 并行节点组 | 取决于子节点 |
| merge | 合并变量或调用组装智能体 | 可选 |
| condition | 规则分支 | 否 |
| human_review | 人工审核 | 是 |
| delivery | 交付产物到外部系统 | 否 |
| end | 流程结束 | 否 |

## 4. 节点协议

每个节点都应遵循统一协议：

```json
{
  "nodeId": "agent_requirement_analysis",
  "nodeType": "agent",
  "name": "需求分析智能体",
  "inputVariables": ["project_info"],
  "outputVariables": ["req_list", "risk_level"],
  "config": {},
  "retryPolicy": {},
  "timeoutSeconds": 300
}
```

## 5. 变量系统

变量必须显式声明，不能让下游节点自由读取任意上游结果。

变量应包含：

- 变量名
- 类型
- 来源节点
- 描述
- JSON Schema
- 是否敏感
- 是否允许作为交付产物

推荐类型：

- `string`
- `number`
- `boolean`
- `object`
- `array`
- `file`
- `decision`

## 6. 智能体节点输出模式

智能体节点支持三种输出模式：

### 6.1 一次性输出

智能体运行一次，直接写入输出变量。

适合：

- 摘要
- 提取
- 分类
- 格式转换

### 6.2 追问模式

智能体可以向用户追问，多轮完成后输出总结结果。

必须配置：

- 最大追问轮次
- 超时时间
- 用户确认动作
- 中途终止策略

### 6.3 内嵌用户输入后继续

智能体先分析，再暂停等待用户确认或补充信息，然后继续运行。

适合：

- 需求确认
- 方案调整
- 审核前补充材料

## 7. 暂停与恢复

底层统一建模：

```text
running -> paused -> waiting_event -> resumed -> running
```

暂停来源：

- 用户输入
- 智能体追问
- 人工审核
- 外部 Webhook 回调
- 高风险 MCP 调用确认

恢复事件必须记录：

- 操作人
- 操作时间
- 输入内容
- 附件
- 审核意见
- 来源渠道

## 8. 并行与合并

并行节点组用于同时执行多个子任务。

完成策略：

- 全部完成后合并
- 任一完成即继续
- 超时跳过未完成项

合并方式：

- 变量透传
- JSON 结构化合并
- 模板组装
- 调用组装智能体

## 9. 条件分支

条件分支只做规则判断，不建议直接让 AI 决定路由。

判断依据：

- 上游变量值
- 智能体输出字段
- 用户输入结果
- 外部 API 返回值

要求：

- 必须配置默认分支。
- 条件表达式要可校验。
- 分支结果要写入执行日志。

## 10. 版本与回放

工作流发布后生成不可变版本。

运行时记录：

- 工作流版本
- 节点配置版本
- 智能体模板版本
- Skill 版本
- MCP 版本
- 模型配置

这样可以保证历史结果可解释、可回放。

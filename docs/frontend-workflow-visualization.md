# 前端工作流展示规范

## 1. 设计目标

Agentum 的前端必须让用户清楚理解一条智能体流程“准备做什么、正在做什么、做完了什么、为什么停住、下一步由谁处理”。

工作流展示不是简单把节点画出来，而是要把 AI 执行过程产品化：

- 看清每个步骤的职责。
- 看清每个步骤使用的智能体、Skills、MCP 和知识库。
- 看清智能体是直接输出，还是允许用户追问确认。
- 看清哪些步骤并行执行，哪些步骤等待合并。
- 看清需要用户输入什么，以及输入会进入哪个变量。
- 看清每个节点完成了哪些工作，产生了哪些结果。
- 看清流程产物如何进入下一步或最终交付。

## 2. 核心视图

建议工作流编辑器和运行详情页都采用“三层信息结构”。

```text
左侧：节点库 / 流程大纲
中间：工作流画布
右侧：节点详情 / 运行详情 / 配置面板
底部：执行日志 / 变量面板 / 工具调用记录
```

### 2.1 流程大纲

流程大纲用于帮助用户快速扫完整条流程，尤其适合节点很多时定位。

每个步骤展示：

- 步骤序号
- 节点名称
- 节点类型
- 当前状态
- 是否需要用户介入
- 是否包含工具调用
- 是否属于并行分组

状态示例：

- 未开始
- 执行中
- 等待输入
- 等待审核
- 已完成
- 已跳过
- 执行失败

### 2.2 工作流画布

画布展示流程结构、分支、并行和合并关系。

画布必须支持两种模式：

- **设计态**：展示配置是否完整、变量是否连通、节点是否可发布。
- **运行态**：展示执行进度、节点状态、耗时、暂停点和错误。

设计态关注“这条流程是否配置正确”。运行态关注“这次执行发生了什么”。

### 2.3 节点详情面板

用户点击任意节点后，右侧面板展示该节点的完整信息。

面板应分为：

```text
基础信息
输入来源
执行配置
能力装配
交互模式
输出产物
运行记录
权限与审计
```

不同节点类型可以隐藏不相关分组，但分组顺序应保持一致。

### 2.4 底部执行面板

底部面板用于展示跨节点的运行证据链。

建议使用 Tabs：

- 执行日志
- 变量快照
- 工具调用
- 用户输入
- 审核记录
- 交付记录

## 3. 节点卡片展示

节点卡片需要在有限空间里表达关键信息，不能只显示节点名称。

### 3.1 所有节点通用信息

所有节点卡片展示：

- 节点名称
- 节点类型图标
- 状态徽标
- 输入变量数量
- 输出变量数量
- 是否暂停点
- 是否有权限限制
- 最近一次耗时

示例：

```text
需求分析智能体
智能体节点 · 已完成 · 32s
输入 1 · 输出 2 · 可追问
```

### 3.2 智能体节点卡片

智能体节点需要额外展示：

- 智能体模板名称
- 已装配 Skills 数量
- 已装配 MCP 数量
- 是否使用知识库
- 输出模式
- 是否允许追问

输出模式展示建议：

- `一次性输出`
- `追问确认`
- `分析后暂停`

卡片示例：

```text
需求分析智能体
Skills 2 · MCP 1 · 知识库 1
模式：追问确认
输出：req_list、risk_level
```

### 3.3 用户输入节点卡片

用户输入节点需要展示：

- 表单字段数量
- 是否允许附件
- 是否必填
- 输出变量
- 等待对象

卡片示例：

```text
补充项目资料
表单 5 项 · 附件可选
等待：发起人
输出：project_info
```

### 3.4 并行节点组卡片

并行节点组需要明确表达并行关系。

展示：

- 子任务数量
- 完成策略
- 超时策略
- 当前完成进度

卡片示例：

```text
并行获取数据
3 个子任务 · 全部完成后合并
进度：2 / 3
```

### 3.5 合并 / 组装节点卡片

合并节点需要展示：

- 接收变量数量
- 合并方式
- 是否挂载组装智能体
- 输出变量

合并方式：

- 变量透传
- JSON 合并
- 模板组装
- 智能体组装

### 3.6 条件分支节点卡片

条件分支节点需要展示：

- 判断字段
- 分支数量
- 默认分支
- 最近命中的分支

条件分支必须让用户一眼看懂为什么走这条路。

### 3.7 人工审核节点卡片

人工审核节点需要展示：

- 审核人或审核角色
- 审核动作
- 当前等待对象
- 是否允许修改内容后继续
- 超时策略

### 3.8 交付节点卡片

交付节点需要展示：

- 交付目标
- 产物来源变量
- 文件类型或消息类型
- 最近交付状态

交付目标示例：

- Word / PDF
- 邮件
- OA 流程
- IM 消息
- Webhook
- 数据库

## 4. 智能体执行展示

智能体执行是 Agentum 的核心差异点，不能只显示“AI 正在运行”。

### 4.1 执行摘要

智能体节点运行后，应展示一段结构化摘要：

- 本节点目标
- 使用了哪些输入
- 调用了哪些工具
- 参考了哪些知识库
- 完成了哪些工作
- 生成了哪些输出变量
- 是否需要用户确认

### 4.2 工具调用展示

每次 MCP 或工具调用都应可查看。

展示字段：

- 工具名称
- 所属 MCP 服务
- 调用时间
- 耗时
- 调用状态
- 参数摘要
- 结果摘要
- 是否命中敏感权限
- 审计编号

敏感参数默认脱敏，不允许前端展示密钥或完整凭证。

### 4.3 Skills 展示

Skills 是智能体能力来源，应在节点详情里可见。

展示：

- Skill 名称
- Skill 版本
- Skill 类型
- 作用说明
- 是否被当前节点启用

### 4.4 知识库展示

如果智能体使用 RAG，需要展示：

- 知识库名称
- 检索关键词或问题摘要
- 命中文档数量
- 引用片段摘要
- 引用来源

## 5. 追问与一次性输出展示

智能体输出模式必须在设计态和运行态都清楚展示。

### 5.1 一次性输出

展示文案：

```text
该节点采用一次性输出，AI 结果会直接写入输出变量并进入下一步。
```

运行态展示：

- 输出内容
- 输出变量
- schema 校验结果
- 是否已进入下一步

### 5.2 追问确认

展示文案：

```text
该节点允许用户追问，用户确认后才会生成最终总结并进入下一步。
```

运行态展示：

- 当前追问轮次
- 最大追问轮次
- 用户历史追问
- AI 历史回答
- 确认按钮
- 终止按钮
- 最终总结变量

### 5.3 分析后暂停

展示文案：

```text
该节点会先生成初步分析，然后暂停等待用户补充或确认。
```

运行态展示：

- AI 初步分析
- 需要用户补充的问题
- 用户输入表单
- 继续执行按钮
- 补充内容写入变量

## 6. 并行执行展示

并行执行必须让用户看清“哪些任务同时跑、哪些已经完成、哪里在等待”。

### 6.1 并行组视觉

建议并行组在画布中使用分组边框或泳道。

并行组头部展示：

- 分组名称
- 子任务数量
- 完成策略
- 超时策略
- 当前进度

### 6.2 子任务状态

每个并行子任务展示：

- 等待中
- 执行中
- 已完成
- 已失败
- 已超时
- 已跳过

### 6.3 合并等待

如果合并节点正在等待并行任务完成，需要明确展示：

```text
正在等待 1 个子任务完成：用户调研智能体
```

不要只显示“运行中”。

## 7. 用户输入展示

用户输入可能出现在独立节点，也可能嵌入智能体节点内部。

### 7.1 输入需求说明

用户输入区域必须展示：

- 为什么需要输入
- 需要输入什么
- 输入会影响哪个节点
- 输入会写入哪个变量
- 不填会发生什么

### 7.2 表单字段

字段类型：

- 文本
- 富文本
- 下拉
- 多选
- 日期时间
- 文件上传
- 人员选择
- 部门选择
- 数字
- 开关

字段必须支持：

- 必填
- 默认值
- 说明文案
- 校验规则
- 示例

### 7.3 暂停提示

流程暂停时，页面顶部或节点附近应展示明确提示：

```text
流程已暂停，正在等待发起人补充项目资料。
```

并展示可操作入口，而不是让用户自己找节点。

## 8. 输出变量展示

变量是工作流前后步骤连接的核心，前端必须显式展示。

### 8.1 节点级变量

每个节点详情展示：

- 输入变量
- 输出变量
- 变量类型
- 变量来源
- 是否敏感
- 示例值

### 8.2 全局变量面板

画布底部提供变量面板，展示当前流程中所有变量。

字段：

- 变量名
- 类型
- 来源节点
- 当前值摘要
- 最近更新时间
- 被哪些下游节点引用

### 8.3 变量引用提示

当某个变量被删除、改名或类型变化时，前端必须提示受影响节点。

## 9. 运行证据链

运行详情页需要形成完整证据链。

每个节点运行记录展示：

- 开始时间
- 结束时间
- 耗时
- 输入快照
- 输出快照
- 工具调用
- 用户操作
- 审核操作
- 错误与重试

证据链的目标不是展示所有原始日志，而是让业务用户和管理员能解释结果从哪里来。

## 10. 信息层级建议

为了避免界面过载，采用渐进披露。

默认展示：

- 节点名称
- 状态
- 模式
- 工具数量
- 输入输出变量
- 当前阻塞原因

点击后展示：

- 完整配置
- 工具调用明细
- Prompt 摘要
- Skills / MCP 版本
- 运行日志

管理员或审计视角展示：

- 完整调用链
- 脱敏参数
- 审计编号
- 权限命中记录
- 重试记录

## 11. 关键交互

必须支持：

- 点击节点查看详情。
- 点击变量查看来源和引用。
- 点击工具调用查看结果摘要。
- 从暂停提示直接跳转到需要处理的节点。
- 并行组展开/折叠。
- 运行态按状态筛选节点。
- 错误节点一键定位。
- 人工审核和用户输入从同一运行详情页处理。

## 12. 设计结论

Agentum 前端的核心不是“把流程画出来”，而是：

- 让用户看懂智能体为什么这样执行。
- 让用户知道哪里需要自己参与。
- 让用户确信 AI 输出如何进入下一步。
- 让管理员能审计工具调用和交付行为。

因此，工作流展示必须围绕“步骤、能力、输入、输出、状态、证据链”六个关键词设计。

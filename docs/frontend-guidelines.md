# 前端设计规范

## 1. 设计目标

Agentum 前端不是营销页面，而是一个专业的企业工作台。

目标：

- 清晰
- 稳定
- 美观
- 高信息密度但不压迫
- 易于长期使用

## 2. 技术选择

推荐：

- React
- TypeScript
- React Flow
- Tailwind CSS
- Radix UI / shadcn/ui
- lucide-react
- TanStack Query
- Zustand
- React Hook Form
- Zod

## 3. 页面结构

推荐核心页面：

- 工作台首页
- 工作流列表
- 工作流编辑器
- 工作流运行详情
- 智能体模板库
- Skills 库
- MCP 服务库
- 知识库
- 交付目标管理
- 审计日志
- 权限管理
- 系统设置

## 4. 画布要求

画布是产品核心，优先保证可读性和稳定性。

要求：

- 节点类型颜色统一。
- 节点尺寸稳定，不因文字长短大幅跳动。
- 连线清晰，选中态明显。
- 支持缩放、拖拽、框选、自动布局。
- 支持节点搜索和快速定位。
- 支持运行态高亮当前节点。
- 支持查看节点输入输出变量。

节点颜色建议：

| 节点类型 | 颜色 |
| --- | --- |
| 触发节点 | 中性灰 |
| 用户输入节点 | 橙色 |
| 智能体节点 | 蓝色 |
| 并行节点组 | 绿色 |
| 合并节点 | 紫色 |
| 条件分支节点 | 黄色 |
| 人工审核节点 | 红色 |
| 交付节点 | 橘红色 |

## 5. 节点卡片

节点卡片必须展示：

- 节点名称
- 节点类型
- 配置摘要
- 输入变量
- 输出变量
- 暂停标记
- 运行状态

不要在节点卡片里堆太多配置项，详细配置放右侧面板。

## 6. 右侧配置面板

节点配置使用右侧抽屉。

推荐结构：

```text
基础信息
输入变量
核心配置
输出变量
权限与审计
高级设置
```

要求：

- 高级设置默认折叠。
- 变量选择器支持点击插入。
- 错误提示要具体。
- 表单保存前必须校验。
- 修改生产流程时要提示影响范围。

## 7. 运行态界面

运行态必须清楚告诉用户当前流程在哪里、为什么停住、下一步谁处理。

应展示：

- 当前状态
- 当前节点
- 节点耗时
- 等待对象
- 暂停原因
- 输入变量
- 输出变量
- 错误信息
- 重试按钮
- 交付物

## 8. 视觉要求

要求：

- 不使用大面积无意义渐变。
- 不用花哨动效掩盖信息层级。
- 卡片圆角保持克制。
- 按钮尺寸和间距统一。
- 图标优先使用 lucide-react。
- 表格、表单、抽屉、弹窗风格统一。
- 移动端可以只保证基础查看，不强求完整画布编辑。

## 9. 文案要求

文案使用中文，简洁具体。

推荐：

- “智能体节点缺少输出变量”
- “当前流程正在等待审核人处理”
- “该 MCP 服务需要管理员授权后才能调用”

不推荐：

- “配置错误”
- “系统异常”
- “操作失败”
